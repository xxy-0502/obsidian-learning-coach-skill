# Personalization

Read `LearningVault/settings/background.md` before teaching or reviewing. If missing, run `scripts/init_vault.py`.

Template:

```markdown
# 学习者背景

## 正在读的年级

## 正在学习的科目、知识等

## 目前遇到的问题

## 学习偏好

## 目标程度

## 例子偏好

## 不喜欢的讲解方式
```

Use the background to tune:

- explanation depth
- terminology density
- example difficulty
- analogy style
- checkpoint difficulty
- pace and amount of scaffolding

Do not copy private background details into Obsidian knowledge notes unless the user explicitly asks.

## Start Assessment Persistence

After the learner answers the start assessment:

- Record topic-specific purpose, target, scope, deadline, and constraints in `progress/[主题]/进度.md`.
- Treat broad preferences such as preferred examples, explanation style, target depth, and disliked teaching methods as candidate global background.
- Ask before writing candidate global background to `LearningVault/settings/background.md`, unless the user explicitly says to update their background.
- Do not write temporary topic goals into `settings/background.md`.
