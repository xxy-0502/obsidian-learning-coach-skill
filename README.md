# Obsidian Learning Coach Skill

一个轻量学习课件 skill。

它的边界很硬：

- AI 生成 `lessons/`：定义、原理骨架、边界、误区、画图任务、自检题。
- 用户生成 `notes/`：自己的解释、类比、图、复述、总结和疑问。
- AI 生成 `checks/` 或聊天反馈：只做事实核查，不润色、不重写。
- AI 通过 `interactions/` 做交互式检查：一题一答，先让用户尝试，再纠偏。
- AI 维护三份轻量状态文件：`学习路线.md`、`错题遗漏.md`、`复习计划.md`。

这个 skill 不追求让 AI 把知识讲到顺滑可读。它追求让用户必须亲手重构理解。

## 适合的请求

```text
我想学 HTTPS，给我课件，不要给我笔记
```

```text
我完全不懂进程和线程，只给定义、原理和自检问题
```

```text
这是我自己写的笔记，请只检查事实错误，不要润色
```

```text
用交互式学习带我过一遍 HTTPS，每次只问我一个问题
```

```text
我想学操作系统，先给我安排进度，然后按进度带我学
```

```text
今天该复习什么？
```

## 默认目录

```text
LearningVault/
  topics/
    [topic]/
      lessons/
      notes/
      checks/
      interactions/
      学习路线.md
      错题遗漏.md
      复习计划.md
```

## lesson 是什么

`lessons/` 是 AI 产物，只放课件原料：

- 前置概念
- 核心定义
- 原理骨架
- 边界条件
- 易错点
- 结构图任务
- 自检问题

## note 是什么

`notes/` 是用户产物。AI 不代写。

用户在这里写：

- 自己的人话解释
- 自己的类比
- 自己画的图
- 自己的复述
- 自己的疑问
- 自己的错因

## fact check 是什么

当用户提交自己的笔记，AI 只检查：

- 定义是否准
- 因果关系是否反
- 图中箭头或顺序是否错
- 公式、单位、边界条件是否错
- 是否漏掉必要条件

如果无错，只回复：

```text
事实无误
```

如果有错，只列错误点，不提供重写版。

## interactive learning 是什么

交互式学习不是 AI 继续讲课，而是 AI 组织检查点：

- 一次只问一个问题
- 你先回答
- AI 只纠正你的答案
- 通过后进入下一题
- 必要时记录到 `interactions/`、`学习路线.md`、`错题遗漏.md` 和 `复习计划.md`

常见题型：

- 复述定义
- 画结构或箭头
- 区分相似概念
- 判断边界条件
- 找出错误说法
- 把原理迁移到一个小例子

## 三个状态文件是什么

三份状态文件都不写知识解释：

- `学习路线.md`：学习路线、当前 lesson、用户 note 状态、下一步动作
- `错题遗漏.md`：预判坑点、真实踩坑、遗漏条件、正确边界
- `复习计划.md`：复习队列、复习记录、下次复看日期

学习时必须先看这三份文件，按 `学习路线.md` 第一条未完成项目继续。

## review 是什么

复习不是重新讲课，而是到期后先考你：

- 先从 `复习计划.md` 的 `复习队列` 找到期项目
- 优先复习 `错题遗漏.md` 里的错误
- AI 先问问题，等你回答
- 答错就记录坑点并安排更近的复看
- 答对再安排更远的复看

可用脚本：

```powershell
python scripts/scan_reviews.py --vault LearningVault --date 2026-06-24
```

添加复看：

```powershell
python scripts/add_review.py `
  --review-plan LearningVault/topics/HTTPS/复习计划.md `
  --date 2026-06-25 `
  --content "HTTPS lesson 01" `
  --reason "首次学习后复看" `
  --pitfall "公钥加密误解"
```

## 不做什么

默认不做：

- 长资料转换
- PDF 拆章
- 知识地图维护
- dashboard
- 重型自动复习系统
- 详细 concept notes
- AI 代写个人笔记

这些功能如果以后需要，可以作为独立 skill 或显式扩展重新加入。
